  •  package config

    Provides main configuration classes.

    Provides main configuration classes.

    Basic usage

    Usual steps are:

    • Load the configuration from an external XML file
    • Add one or more configuration spaces with mapped statements
    • Create the persistenceContext

    Code sample 

    val config = Configuration("META-INF/mybatis.xml")
config.addSpace("ns") { space =>
  space ++= MyDAO
}
val persistenceContext = config.createPersistenceContext

    Another example without external XML file and with default namespace 

    val config = Configuration(
 Environment(
   "default",
   new JdbcTransactionFactory(),
   new PooledDataSource(
     "org.hsqldb.jdbcDriver",
     "jdbc:hsqldb:mem:scala",
     "sa",
     ""
   )
 )
)
config ++= MyDAO

val persistenceContext = config.createPersistenceContext

  9.  def cache(impl: T[_ <: Cache] = DefaultCache, eviction: T[_ <: Cache] = Eviction.LRU, flushInterval: Long = -1, size: Int = -1, readWrite: Boolean = true, blocking: Boolean = false, props: Properties = null): ConfigurationSpace

    Adds cache support to this space.

    Adds cache support to this space.

    impl

    Cache implementation class

    eviction

    cache eviction policy (LRU,FIFO,WEAK,SOFT)

    flushInterval

    any positive integer in milliseconds. The default is not set, thus no flush interval is used and the cache is only flushed by calls to statements.

    size

    max number of objects that can live in the cache. Default is 1024

    readWrite

    A read-only cache will return the same instance of the cached object to all callers. Thus such objects should not be modified. This offers a significant performance advantage though. A read-write cache will return a copy (via serialization) of the cached object, this is slower, but safer, and thus the default is true.

    props

    implementation specific properties.
